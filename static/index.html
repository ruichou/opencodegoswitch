首次使用时，点击下方按钮自动写入 Codex 配置。

或手动复制以下配置到 ~/.codex/config.toml： 

model_provider = "opencode-go-switch"
model = "deepseek-v4-pro"
model_reasoning_effort = "medium"
disable_response_storage = true
network_access = "enabled"
windows_wsl_setup_acknowledged = true

[model_providers.opencode-go-switch]
name = "OpenCode Go (via Switch)"
base_url = "http://127.0.0.1:7878/v1"
wire_api = "responses"
requires_openai_auth = true

[features]
js_repl = false

[desktop]
followUpQueueMode = "queue"

首次使用时，点击下方按钮生成配置脚本。

方案 A：运行生成的脚本 (推荐)

点击上方按钮后，桌面会生成 claude-code-opencode.ps1，在 VS Code 终端中执行即可。

方案 B：手动设置环境变量

# Windows PowerShell
$env:ANTHROPIC_BASE_URL="http://127.0.0.1:7878/v1"
$env:ANTHROPIC_API_KEY="opencode-go-switch"

注意：MiniMax / Qwen 模型走 Anthropic 直通，DeepSeek / GLM / Kimi / MiMo 走自动格式转换。

POST http://127.0.0.1:7878/v1/responses → Codex
POST http://127.0.0.1:7878/v1/messages → Claude Code
POST http://127.0.0.1:7878/v1/chat/completions → 直通 Chat API
GET http://127.0.0.1:7878/v1/models → 模型列表
